# Fast No Slow Rules

## Core Principles
- HR is the primary metric
- Simplicity over feature expansion
- Real-time clarity during runs is critical

## UX Rules
- UI must be readable at a glance while moving
- Avoid clutter or excessive data
- Prioritize clear feedback over more information

## Coaching Rules
- Focus on actionable feedback
- Avoid overwhelming the runner
- Prefer subtle, consistent guidance

## Non-Negotiables
- Do not add new features during evaluation
- Improve clarity, not complexity
